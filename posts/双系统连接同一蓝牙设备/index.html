    双系统连接同一蓝牙设备， Windows 10, Ubuntu 21.10

    问题描述#

    当安装双系统时，切换系统需要重新移除蓝牙然后重新配对才能正常使用的情况。 网上的解决方案有很多，但是随着系统的不断更新，方法也需要不断的更新，目前使用的方法发现要更简单一些。

    导致的原因#

    蓝牙设备连接时设备会存储连接 id，电脑和蓝牙音响都会保存这个连接id，但是连接 id 是随机生成的，两个系统会生成各自的连接 id。

    解决#

    只需要要将两个蓝牙的连接 id 保持一致就行。本次是 Ubuntu 修改成和 Windows 一样的连接 id，反过来也是生效的

    步骤#

    • 分别在两个系统上连接这个蓝牙设备，主要是为了生成连接信息，后面会修改
    • 在 windows 上查看连接 id
    • 将 Winodows 上的连接 id 复制到 Ubuntu 下
    • 重启电脑

    Windows 查看蓝牙连接 id#

    1.蓝牙的连接 id 是加密的，需要使用 psexec.exe 工具，下载 ZIP 后解压。

    使用管理员权限打开 cmd 并到 cd 到解压目录执行:

    psexec64.exe -si regedit

    2.查看蓝牙设备 id:

    设备-> 设备和打印机 -> 右键蓝牙设备 -> 属性 -> 蓝牙 -> 唯一标识符

    另一种方法是到设备管理中找到对应的蓝牙设备属性，在下拉选项中有设备 id 信息也可以看到

    3.在注册表的路由地址

    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\BTHPORT\Parameters\Keys\<本机蓝牙 MAC>\<蓝牙设备 id>

    双击打开后会看到两行字符串，无法复制，可以手动抄下来，记得取消空格

    4.将 Windows 下的连接 id 复制到 Ubuntu 下

    重启进入 Ubuntu 系统，使用 sudo 权限修改蓝牙的连接信息

    # 切换到 sudo 权限
sudo su

# 到设备蓝牙目录下
cd /var/lib/bluetooth/<本机蓝牙 id>/<设备蓝牙 id>/

# 先备份 info
cp info infoback

# 修改
vim info

    原始内容类似

    [General]
Name=JBL GO
Class=0x240404
SupportedTechnologies=BR/EDR;
Trusted=true
Blocked=false
Services=00001101-0000-1000-8000-00805f9b34xx;00001108-0000-1000-8000-00805f9b34xx;0000110b-0000-1000-8000-00805f9b34xx;0000110c-0000-1000-8000-00805f9b34xx;0000110e-0000-1000-8000-00805f9b34xx;0000111e-0000-1000-8000-00805f9b34xx;

[LinkKey]
Key=36AAAE69BBD2204A5A81255E9E4C1C00
Type=4
PINLength=0

    修改 [LinkKey] 下的 Key 成为 Windows 抄下来的连接 id（去除空格，共 32 位）

    修改后直接重启系统，两个系统都检查一下是否是直接连接上的
